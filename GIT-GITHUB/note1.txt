se placer dans un dossier puis faire 

git init :: crée un nouveau repo en local
git add nom_fichier [ou dossier] :: ajoute un fichier à la liste des fichiers à versionner
git commit -m "Commentaire du commit" :: effectue un commit
git commit -a -m "Commentaire du commit" :: effectue un commit sur un [des] fichiers déjà ajouté
git log :: affiche la liste des commit éffectués par ordre récent
git checkout CodeSHAduCommit :: se positioner sur un commit
git checkout master :: se position sur dernier commit
git revert CodeSHAduCommit :: Annuler un commit
git commit --amend -m "Nouveau commentaire" :: Mettre à jour le commentaire d'un commit
git reset --hard :: Annuler tous les changements non encore commité

git clone LienDuGit :: télécharge un git (repo) à partir d'un site distant en local
git push origin master :: upload un repo sur un site distant (site d'origine)
git pull origin master :: met à jour son repo local avec les commit existant en remote mais pas en ocal

git branch :: Pour voir les branches présentes dans votre repo
git branch nouvelle-branche :: Pour créer une nouvelle branche
git checkout nouvelle-branche

git checkout -b ma-branche :: pour créer une branche et vous y positionner

git merge brancheB : ajouter  dans une branche A les mises à jour que vous avez faites dans une autre branche B. (cette commande suppose que vous êtes positionné dans la brache A)

git blame nomdufichier.extension :: retrouver qui a fait une modification particulière dans un fichier
git show debut_du_sha :: renvoie directement les détails d'un commit


git stash :: mettre de côté des modifications en cours...
git stash pop :: reprendre des modifications mises de côté.
git stash apply :: garder les modifications dans votre stash


 pull request : proposer une modification à un projet hébergé sur GitHub.

Contribuez à des projets open source
	- Tout d'abord, faites un fork du repo auquel vous souhaitez contribuer
	- Ensuite, clonez votre copie depuis GitHub sur votre machine
	- créez une nouvelle branche où vous allez faire vos modifications et placez-vous dedans.
	- Faites vos modifications dans la nouvelle branche et "committez"-les dans Git
	- Envoyez vos modifications sur GitHub en faisant un git push de votre nouvelle branche
	- Faire un puul Request (placez-vous sur votre fork GitHub, sur votre nouvelle branche, et cliquez sur "Compare & pull request").
	
	
    	Fork it
    	Create your feature branch (git checkout -b my-new-feature)
    	Commit your changes (git commit -am 'Add some feature')
    	Push to the branch (git push origin my-new-feature)
    	Create new Pull Request
